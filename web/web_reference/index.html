Q&A

Questions, answered.

The things developers ask before their first request.

Is wavydance.ai compatible with the OpenAI SDK?

Yes — for chat and vision models the API is fully OpenAI-compatible. Point your existing SDK at our base_url and switch providers by changing the model string. Image and video go through the unified async task API instead.

How does billing work?

Pay-as-you-go per token (or per generation for media models), aggregated into one monthly bill across all providers. No seats, no minimums. Budgets and alerts are built in.

What happens when a provider goes down?

Latency-aware routing detects the failure and retries your request on the next-best provider automatically — same model family where possible. You see one consistent endpoint; the failover is invisible.

Do you store or train on my prompts?

No. Requests are relayed, metered and discarded. Only usage metadata (token counts, latency, model) is retained for analytics and billing. Enterprise plans support zero-log mode.

How do I get results from long-running image / video jobs?

Two ways: register a webhook and we push the result the moment the task succeeds, or poll GET /v1/tasks/:id at your own pace. Both return the same task object.
